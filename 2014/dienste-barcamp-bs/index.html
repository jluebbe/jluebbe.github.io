GitHub

  • GitLab statt GitHub
    • Repo-Verwaltung
    • Wiki
    • Issue-Tracking

Mail

  • Postfix, Dovecot, Roundcube, Mailman, Amavis
  • Viel Aufwand! → am besten für viele Nutzer
    • Spam
    • Fehlkonfigurationen führen zu Mailverlust!
  • Oder fertig vorbereitet: z.B. Kolab oder SOGo

DNS & DynDNS

  • Bind 9
    • DNSSEC, DANE
  • DynDNS: nsupdate.info / stratum0.net
  • Mit jemand anderem "secondaries tauschen"

XMPP (Jabber)

  • (Gruppen-)Chat
  • Federated
  • Android-Client: Conversations (OTR und GPG mit OpenKeychain) oder ChatSecure
  • Server: prosody oder ejabberd

Video-/Audio-Konferenz

  • Multi-User-Voice-Chat: Mumble
  • Video-Konferenz im Browser: palava.tv
  • VoIP: ostel.co (oder z.B. asterisk)

VPN

  • Sicherer Netzzugang von überall
  • OpenVPN über TCP oder UDP
  • iodine als DNS-Tunnel

"Cloud"

  • OwnCloud: DropBox, Kalender, Adressen, ...
  • Etherpad: Multiuser-Texteditor
  • ZeroBin: Verschlüsselter Textsnipsel-Austausch
  • Social: Friencia, Diaspora, pump.io

Virtualisierung

  • Als Basis für die anderen Dienste
  • Physikalischen Server gemeinsam nutzen
  • libvirt (KVM oder Xen)
    • virt-manager als GUI per SSH
    • kimchi als Web-Interface

Wie?

  • Jeder betreibt 1+ Dienste
  • Gemeinsames (virtualisiertes) Hosting
  • Es gibt viele gut Anleitungen
  • Oder: Im nächsten Hackerspace mitmachen
